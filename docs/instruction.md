# コード規約

## UI実装

- デザインはこちらから指示するので、UIの実装はデザイン指示を受けてから行うこと。実装品質は商用レベルを維持すること。
- UIは必要最低限のレイアウトにとどめ、カスタムアニメーションや複雑なウィジェットツリーは避けること。
- パッケージが提供するUIコンポーネントを積極的に活用し、独自実装でカバーしようとしないこと。

## フォルダ構成

- 複数画面で使い回せるウィジェットは `lib/core/widgets/` に切り出す。プライベートクラス（`_` プレフィックス）が汎用的な形になっていたら共通化を検討する。
- `lib/core/widgets/` は種類別にサブディレクトリで整理する。

  | ディレクトリ | 用途 |
  |---|---|
  | `buttons/` | ボタン系ウィジェット |
  | `indicators/` | ローディングなど状態表示系 |
  | `navigation/` | ナビゲーション系 |

- ウィジェットではなく関数・ヘルパーは `lib/core/utils/` に置く（例：`showErrorSnackBar`）。

## デザイントークン

- 色・サイズの直書き禁止。必ず定数として定義して使う。

  | クラス | 用途 |
  |---|---|
  | `AppColors` | 色 |
  | `AppDimens` | サイズ・余白 |
