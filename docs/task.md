# 実装タスク

粒度粗め。順番通りに進めることを推奨。

---

## 1. データモデル・リポジトリ基盤

後続の全機能が依存するので最初に固める。

- `AppUser` / `WalkSession` / `Spot` モデルクラスの定義
- Firestore リポジトリの実装（CRUD）
- Firebase Storage リポジトリの実装（写真アップロード）

---

## 2. 初回ログイン時のユーザー登録

サインイン直後に `users/{uid}` を Firestore に作る処理。
現状の `signInWithGoogle()` に追記する形で実装。

---

## 3. 散歩セッション（コア機能）

アプリのメイン機能。ここが動けば一番「使えてる感」が出る。

- セッション開始・終了
- GPS で `routePoints` を定期取得して蓄積
- セッション終了時に Firestore へ一括保存
- セッション中画面の UI（経過時間・距離・ルート描画）

---

## 4. スポット記録

セッション中に呼び出す機能なので 3 の後。

- 写真撮影 / カメラロール選択
- Firebase Storage へアップロード
- `spots` コレクションに保存
- スポット記録画面の UI

---

## 5. 地図画面（他ユーザーのデータ表示）

1〜4 でデータが揃ってから実装する。

- `walkSessions` からルートをポリラインで描画
- `spots` をマーカーで表示（リアルタイム）
- マーカータップでスポット詳細ポップアップ
- 現在地ボタン

---

## 6. 記録一覧・詳細

- 散歩記録のリスト表示（ページネーション）
- 絞り込み（自分のみ・日付）
- 記録詳細画面（地図 + スポット一覧）
- 自分の記録：公開 / 非公開切り替え・削除

---

## 7. プロフィール

- ユーザー情報表示・編集（表示名・アバター）
- 統計（総散歩回数・距離・時間）
- サインアウト
